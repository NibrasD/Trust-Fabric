You are an expert full-stack blockchain developer specializing in Stellar, Soroban smart contracts, and AI agent systems.

Build a complete open-source project called **"Stellar Agent Trust Fabric"** for the Stellar Agents x402 Hackathon.

### Project Overview
Stellar Agent Trust Fabric is a trust and permission layer for AI agents on Stellar. It enables autonomous agents to:
- Discover and pay for services using **x402 micropayments** (and optionally MPP).
- Operate with **scoped, temporary, and limited permissions** (no unlimited wallet access).
- Build and maintain an **on-chain reputation score** based on payment history and post-transaction ratings (no lending, no interest, no staking, no slashing).

Core features (no DeFi, no loans, no yield):
1. **Scoped Session Permissions**: Agents get temporary "session keys" or contract account authorizations that allow:
   - Maximum spend limit (e.g., 0.5 USDC per session).
   - Specific allowed endpoints only.
   - Time-limited (e.g., 1 hour or one task).
   - Everything enforced via Soroban smart contracts.

2. **On-Chain Reputation System**:
   - Each agent (identified by contract account or public key) has a reputation profile.
   - Reputation increases based on successful x402 payments + rating (1-5 stars) from the service provider after delivery.
   - Weighted by payment amount (higher value transactions = more weight).
   - Stored on Soroban with simple mapping (agent_address → score, total_transactions, avg_rating).

3. **Agent Discovery Marketplace** (light version):
   - A simple registry where service providers list their x402-protected endpoints with price and category.
   - Agents can query the registry and get results sorted by reputation + price.

4. **Integration with x402 and MPP**:
   - All payments must go through real x402 flow (402 response → payment → access).
   - Support USDC (or any SAC token) via Stellar.

### Technical Stack (must use):
- **Backend**: Node.js + Express (or TypeScript)

- **Stellar SDK**: @stellar/stellar-sdk + Soroban client
- **Deployment**: Testnet first, with clear instructions for mainnet
